IOT protocols
=============

For identification and security, assume:

 * the use of text string pass phrases of up to 35 characters + the \0 character.
 
 * a device can be configured and registered through a phone app using bluetooth.
 
 * each device has a unit idenfification string or code (i.e. something like a
   MAC address.)
 
 * when a device is registered with an account:
 
   - the device is assigned its own secret pass phrase.
   
   
   - the device's id and pass phrase are stored with the account info on the
     server in plain text (i.e. we assume the data is stored securly in a
     database system).  
   
   
When a device connects to the server:

 * it sends its id string 

   


 
  * each account has a secret pass phrase.

   - the account's pass phrase is stored in the device.
