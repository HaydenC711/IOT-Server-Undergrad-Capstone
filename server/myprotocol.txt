Device Authentication
----------------
-device sets opcode to 100 snd sends devId, type, hashed passPhrase
- the passphrase is read from (struct or txt) and hashed
	- if hashed passphrases are the same, server continues
	- otherwise it disconnects
- server checks account struct and finds the account associated with deviceId
		if not found server disconnects device
- AccountNo is stored in the clientInfo struct
